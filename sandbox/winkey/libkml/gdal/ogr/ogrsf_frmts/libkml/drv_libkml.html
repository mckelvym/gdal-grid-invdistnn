Libkml Driver (.kml .kmz)

datasource

LIBKML_USE_DOC.KML
yes

Style

Style tables are wrote to the document in a .kml file, style/style.kml in a kmz file.

Schema

layer

Style

Layer style tables cannot be read or written to a kml layer that is a folder, otherwise they are in the document that is the layer.

Schema

Other

description, timestamp, timespan, open, and Visibility are not supported on layers as there is no place in ogr to store this data.

fields

LIBKML_NAME_FIELD
Name of the OGR field to use for a placemark's name, Defaults to Name
LIBKML_DESCRIPTION_FIELD
Name of the OGR field to use for a placemark's description, Defaults to description
LIBKML_TIMESTAMP_FIELD
Name of the OGR field to use for a placemark's timestamp, Defaults to timestamp
LIBKML_BEGIN_FIELD
Name of the OGR field to use for a placemark's timespan begin, Defaults to begin
LIBKML_END_FIELD
Name of the OGR field to use for a placemark's timespan end, Defaults to end
LIBKML_ALTITUDEMODE_FIELD
Name of the OGR field to use for a geometry's altitudeMode, Defaults to altitudeMode
LIBKML_TESSELLATE_FIELD
Name of the OGR field to use for a polygon's tessellate, Defaults to tessellate
LIBKML_EXTRUDE_FIELD
Name of the OGR field to use for a geometry's extrude, Defaults to extrude
LIBKML_VISIBILITY_FIELD
Name of the OGR field to use for a placemark's visibility, Defaults to visibility
